# Seedance Studio 功能性联通测试计划

> 版本: 1.1 | 日期: 2026-03-29
> 目标: 在浏览器中进行真实端到端联通测试，验证前后端全链路可用性，发现并记录问题

---

## 一、测试环境准备

### 1.1 服务启动检查清单

| 序号 | 检查项 | 命令/操作 | 预期结果 |
|------|--------|-----------|----------|
| 1 | Python 后端启动 | `python server.py` | `Server running on http://127.0.0.1:8787` |
| 2 | Next.js 前端启动 | `npm run dev`（在项目根目录） | `Ready on http://localhost:3001` |
| 3 | MySQL 数据库运行 | 后端启动日志无 DB 连接错误 | 无报错，或回退到 JSON 存储 |
| 4 | `.local-secrets.json` 配置 | 检查 `api_key` 非空 | 文件存在且含有效 API key |
| 5 | 代理转发验证 | 浏览器访问 `http://localhost:3001/api/config` | 返回 JSON 配置（`hasApiKey: true`） |

### 1.2 测试素材准备

为测试图生视频、首帧控制等模式，提前准备以下文件：

| 素材 | 要求 | 用途 |
|------|------|------|
| `test_source.png` | 任意清晰图片，≤ 2MB | TC-03d 图生视频 |
| `test_first_frame.png` | 视频首帧截图或设计图 | TC-03b 首帧控制 |
| `test_last_frame.png` | 视频尾帧截图或设计图 | TC-03c 首尾帧控制 |

> 如果没有现成素材，可用任意风景/动漫截图代替，目的是验证上传和转 base64 链路。

### 1.3 测试工具

- **浏览器**: Chrome DevTools（Network + Console 面板常开）
- **API 调试**: 浏览器 Network 面板 或 curl
- **录屏**: 建议录屏记录测试过程，方便回溯

### TC-01: 代理转发基础联通

**优先级**: P0（阻塞性）
**目的**: 验证 Next.js → Python 后端的代理转发是否正常

**步骤**:

1. 浏览器打开 `http://localhost:3001`
2. 打开 DevTools → Network 面板
3. 观察页面加载时是否自动发起 `/api/config` 请求
4. 观察请求是否返回 200

**验证点**:

| 检查项 | 预期 | 实际 | 通过? |
|--------|------|------|-------|
| `/api/config` 返回 200 | 200 OK | | |
| 响应体包含 `hasApiKey` 字段 | `true` | | |
| 响应体包含 `defaultModel` | `"doubao-seedance-2.0"` | | |
| 响应体包含 `modelHints` | 含 resolution/ratio/duration | | |
| 无 CORS 错误 | Console 无 CORS 报错 | | |

**失败排查**:
- 502/504 → Python 后端未启动
- CORS 错误 → `next.config.ts` rewrites 未生效，重启 Next.js dev server
- `hasApiKey: false` → `.local-secrets.json` 中 `api_key` 为空

### TC-02: 用户认证全流程

**优先级**: P0
**目的**: 验证注册 → 登录 → 会话保持 → 登出 完整链路

**步骤**:

1. 浏览器打开 `http://localhost:3001/login`
2. **注册**: 填写用户名 `testuser_<timestamp>`、密码 `Test1234`，点击注册
3. **登录**: 使用刚注册的账号登录
4. **会话验证**: 登录后检查是否跳转到主页，且 Header 显示用户名
5. **API 认证**: 在 Network 中观察后续请求是否携带 `sid` cookie
6. **登出**: 点击登出按钮
7. **登出验证**: 访问需要认证的页面（如工作流页面），应被重定向到登录页

**验证点**:

| 检查项 | 预期 | 实际 | 通过? |
|--------|------|------|-------|
| 注册 `/api/auth/register` 返回 200 | 200 + `{ok: true}` | | |
| 登录 `/api/auth/login` 返回 200 | 200 + 设置 `sid` cookie | | |
| 登录后 `/api/auth/me` 返回用户信息 | `{id, username}` | | |
| 后续请求自动携带 `sid` cookie | Request Headers 含 `Cookie: sid=...` | | |
| 登出 `/api/auth/logout` 清除 cookie | 200 + cookie 过期 | | |
| 登出后访问受保护 API 返回 401 | 401 Unauthorized | | |

**失败排查**:
- 注册返回 400 → 用户名已存在，换一个
- 登录后无 cookie → 检查后端 `Set-Cookie` 响应头，可能 `SameSite` / `Domain` 问题
- `/api/auth/me` 返回 401 → cookie 未正确传递，检查跨域配置

### TC-03: 视频生成全模式验证

**优先级**: P0
**目的**: 验证全部 6 种视频生成模式的端到端链路
**前置**: 已完成 TC-02 登录

> **费用说明**: 每次生成消耗真实 API 额度。建议 TC-03a 必测（验证核心链路），TC-03b~03f 根据需要选测。
> 全部使用 **480p / 5 秒** 最低配置以控制费用（每次约 ￥2-5）。

#### TC-03a: 纯文字生成（text） ★ 必测

**步骤**:

1. 在主页找到 **提示词输入框**（MainInput 组件）
2. 输入简短测试提示词：`一只小猫在草地上奔跑，阳光明媚`
3. 确认生成参数：
   - 模式: `文字生成`（text）
   - 分辨率: `480p`
   - 比例: `16:9`
   - 时长: `5秒`
   - 镜头: `auto`
   - 运动速度: `steady`
   - 生成音效: `关闭`
4. 点击 **生成** 按钮
5. 观察 Network 面板中的请求

**验证点 — 请求构造**:

| 检查项 | 预期 | 实际 | 通过? |
|--------|------|------|-------|
| `POST /api/tasks` 发出 | 请求体含 `model`, `content`, `resolution` | | |
| `content[0].type` = `"text"` | 文本内容包含用户提示词 | | |
| `content` 数组**仅含**一个 text 项 | 无 `image_url`/`video_url`/`task_id` | | |
| `model` = `"doubao-seedance-2.0"` | 与配置一致 | | |
| `resolution` / `ratio` / `duration` | `"480p"` / `"16:9"` / `5` | | |
| `generate_audio` | `false` | | |
| 响应状态 200 | 返回 `{id, status}` | | |
| 返回 `id` 格式 | `cgt-yyyyMMddHHmmss-xxxxx` | | |

**验证点 — 轮询与状态流转**:

| 检查项 | 预期 | 实际 | 通过? |
|--------|------|------|-------|
| 前端自动发起 `GET /api/tasks/{id}` | 每 2-5 秒轮询一次 | | |
| 状态依次变化 | created → queued → running → succeeded | | |
| `progress_percent` 逐步增长 | 10% → 20% → 60%+ → 100% | | |
| UI 显示进度条 | TaskProgress 组件实时更新 | | |
| 轮询总耗时 | 2-5 分钟内完成 | | |

**验证点 — 完成与下载**:

| 检查项 | 预期 | 实际 | 通过? |
|--------|------|------|-------|
| 最终状态 `succeeded` | `status: "succeeded"` | | |
| 响应含 `content.video_url` | 远程视频 URL 非空 | | |
| 自动下载触发（`auto_save=true`） | `local_asset` 字段出现 | | |
| `local_asset.local_url` 格式 | `/media/videos/cgt-xxx.mp4` | | |
| 本地视频可播放 | 浏览器访问 local_url 返回 200 + video/mp4 | | |
| UI 显示视频预览/播放器 | 视频缩略图可见，点击可播放 | | |
| `proxy_meta.cost` 有值 | 含费用信息 | | |

**失败排查**:
- `POST /api/tasks` 返回 502 → 上游 API (`zlhub.xiaowaiyou.cn`) 不可用
- `POST /api/tasks` 返回 400 → payload 格式错误，在 Console 中打印 `buildPayload()` 输出排查
- 轮询超过 5 分钟仍 `running` → 上游可能超时，检查后端 `PROXY_TIMEOUT` (默认180s)
- `local_asset` 不出现 → 检查 `.local-secrets.json` 中 `auto_save: true` + `storage/videos/` 目录存在且可写
- 视频 URL 403 → 远程 URL 已过期（正常），应通过 `local_url` 访问

## 五、已知风险与注意事项

1. **API 费用**: 视频生成消耗真实费用（480p/5秒 约 ￥2-5/次），批量生成注意控制数量
2. **上游依赖**: 上游 API (`zlhub.xiaowaiyou.cn`) 如果宕机，视频和图片生成无法执行
3. **AI Chat 依赖**: `peiqian.icu` 如果不可用，剧本生成无法执行
4. **MySQL 依赖**: 如果 MySQL 未运行，部分数据可能回退到 JSON 存储，行为可能不一致
5. **中文路径**: 项目路径含中文字符（`oii前端`），某些工具可能有兼容问题
6. **端口冲突**: 确保 8787（后端）和 3001（前端）端口未被占用
7. **Pipeline 耗时**: 四阶段全自动可能需要 10-30 分钟，需耐心等待
