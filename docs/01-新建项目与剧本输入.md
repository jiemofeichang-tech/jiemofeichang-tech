# 01 — 新建项目与剧本输入

> 工作流 Step 1：用户新建漫剧项目，输入剧本文本和参考素材。

---

## 一、流程图

```
┌──────────────────────────────────────────┐
│           用户点击「新建项目」              │
└──────────────┬───────────────────────────┘
               ▼
┌──────────────────────────────────────────┐
│  填写项目信息:                             │
│  ├─ 项目名称（可选，AI 自动生成）           │
│  ├─ 剧本文本（必填，200~50000 字）         │
│  └─ 参考图片（可选，最多 5 张）             │

               ▼
┌──────────────────────────────────────────┐
│  POST /api/projects                       │
│  → 创建项目目录 storage/projects/{id}/     │
│  → 初始化 project.json (status: "draft")   │
│  → 返回项目 ID                             │
└──────────────┬───────────────────────────┘
               ▼
          进入 Step 2 →
```

## 二、用户界面要素

### 2.1 入口

现有入口已实现：
- 首页 `ProjectsPage.tsx` 的「新建项目」按钮
- 导航到 `/workflow/[projectId]/ai-produce`

### 2.2 输入表单

| 字段 | 类型 | 必填 | 校验规则 | 现有状态 |
|------|------|------|---------|---------|
| 项目名称 | `input text` | 否 | ≤50字符；空则从剧本首行/AI摘要自动生成 | ✅ 已有 |
| 剧本文本 | `textarea` | 是 | 200~50000 字；去除首尾空白后校验 | ✅ 已有 |
| 参考图片 | `file upload` | 否 | PNG/JPG/WebP；单张 ≤5MB；最多 5 张 | ⚠️ 需补充 |

### 2.3 剧本输入引导

在 textarea 上方或 placeholder 中提供格式提示：

```
提示：你可以自由输入故事，也可以按以下格式提供更详细的信息：

【故事背景】时代、地点、世界观设定
【主要角色】角色名 — 外貌特征 — 性格特点
【剧情梗概】故事的起承转合
【对白片段】（可选）关键对话内容

支持输入 200~50000 字。越详细，AI 生成的效果越好。
```

### 2.4 快速模板

提供几个一键填入的剧本模板：

| 模板名 | 字数 | 类型 |
|--------|------|------|
| 霸总甜宠 | ~500字 | 都市言情 |
| 修仙逆袭 | ~500字 | 古风玄幻 |
| 校园暗恋 | ~400字 | 青春校园 |
| 末世求生 | ~500字 | 科幻冒险 |

## 六、校验规则

| 规则 | 处理方式 |
|------|---------|
| 剧本为空 | 前端 disable 提交按钮 + toast 提示 |
| 剧本 < 200字 | 前端 warning：「剧本内容较少，建议补充更多细节」 |
| 剧本 > 50000字 | 前端 error：「剧本过长，请精简至 50000 字以内」 |
| 图片格式错误 | 前端拦截，只允许 PNG/JPG/WebP |
| 图片 > 5MB | 前端自动压缩或拒绝 |
| 项目名已存在 | 允许重名（以 ID 区分） |

## 七、版本历史

Step 1 **不记录版本历史**。原因：

- 项目名称和剧本文本属于用户原始输入，修改操作是直接编辑而非 AI 重新生成
- 用户可以随时通过 `PUT /api/projects/{id}` 手动修改项目名和剧本内容
- 不存在需要回退的 AI 生成结果
