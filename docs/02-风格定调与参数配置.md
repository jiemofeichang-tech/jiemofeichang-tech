## 三、影片参数

### 3.1 参数面板

| 参数 | UI 控件 | 选项 | 默认值 | 说明 |
|------|---------|------|--------|------|
| **故事时长** | 按钮组 | `15s` / `30s` / `60s` / `120s` / 自定义 | `60s` | 决定总分镜数 |
| **影片比例** | 按钮组 | `16:9` / `9:16` / `1:1` | `9:16` | 竖屏短剧为主 |
| **对白语言** | 下拉选择 | 中文 / 英文 / 日文 / 韩文 / 无对白 | 中文 | 影响 TTS 和字幕 |
| **单镜头时长** | 滑块 | `3~8s`，步进 1s | `5s` | 自动计算分镜数 |

### 3.2 比例专属构图提示词

```typescript
export const ASPECT_RATIO_PROMPTS: Record<string, string> = {
  "16:9": "widescreen cinematic composition, horizontal storytelling, rule of thirds, cinematic letterbox framing",
  "9:16": "vertical composition, portrait framing, mobile-optimized layout, subject centered or slightly above center, close framing for emotional impact",
  "1:1": "square composition, centered subject, balanced symmetry, social media optimized, bold graphic framing",
};
```

### 3.3 自动分镜数计算

```
分镜数 = Math.ceil(总时长 / 单镜头时长)

例: 60s / 5s = 12 个分镜
    30s / 4s = 8 个分镜
    120s / 5s = 24 个分镜
```

前端在参数面板下方实时显示：`预计生成 {n} 个分镜`

## 五、提示词合成逻辑

### 5.1 合成公式

当用户确认所有选择后，系统自动合成总风格提示词：

```typescript
function compileStylePrompt(config: {
  storyType: string;
  artStyle: string;        // 风格子类名称
  aspectRatio: string;
  durationSec: number;
  shotDurationSec: number;
  language: string;
}): { stylePrompt: string; negativePrompt: string } {

  // 1. 短片类型提示词
  const typePrompt = STORY_TYPE_PROMPTS[config.storyType].prompt;

  // 2. 风格子类提示词
  const style = styleItems.find(s => s.name === config.artStyle);
  const artPrompt = style?.prompt || "";
  const artNegative = style?.negativePrompt || "";

  // 3. 比例构图提示词
  const ratioPrompt = ASPECT_RATIO_PROMPTS[config.aspectRatio];

  // 4. 全局质量锚定词（固定不变，用户不可见）
  const qualityAnchor = "masterpiece quality, professional animation production, consistent art style throughout all shots, cinematic composition, professional color grading, high detail";

  // 5. 通用反面词
  const globalNegative = "low quality, blurry, deformed, extra limbs, bad anatomy, watermark, text, logo, signature, cropped, worst quality, jpeg artifacts";

  // 合成
  const stylePrompt = [artPrompt, typePrompt, ratioPrompt, qualityAnchor].filter(Boolean).join(", ");
  const negativePrompt = [artNegative, globalNegative].filter(Boolean).join(", ");

  return { stylePrompt, negativePrompt };
}
```

### 5.2 合成结果存储

### 5.3 高级编辑：手动修改合成提示词

系统合成的 `compiled_style_prompt` 默认折叠不可见。高级用户可展开直接编辑合成后的提示词和反面提示词。

#### UI 交互

│  [▶ 展开高级编辑]                                   │

      点击展开后 ↓

┌──────────────────────────────────────────────────┐

│  ┌────────────────────────────────────────────┐  │
│  │ Studio Ghibli style, soft watercolor       │  │
│  │ textures, warm natural lighting, ...       │  │
│  │ (可自由编辑上方内容)                         │  │
│  └────────────────────────────────────────────┘  │
│                                                    │
│  ✏️ 反面提示词                                      │
│  ┌────────────────────────────────────────────┐  │

│  [确认修改]  [取消]                                  │
└──────────────────────────────────────────────────┘
```

#### 数据模型变更

```typescript
interface StyleConfig {
  // ... 现有字段 ...
  prompt_manually_edited: boolean;     // 🆕 是否被手动编辑过
  prompt_edit_history: string[];       // 🆕 编辑前的原始值（用于"还原默认"）
}
```

#### 前端逻辑

```typescript
function handlePromptEdit(newPrompt: string, newNegative: string) {
  const config = project.style_config!;
  // 首次编辑时保存原始版本
  if (!config.prompt_manually_edited) {
    config.prompt_edit_history = [config.compiled_style_prompt];
  }
  config.compiled_style_prompt = newPrompt;
  config.compiled_negative_prompt = newNegative;
  config.prompt_manually_edited = true;
  wfUpdateProject(project.id, { style_config: config });
}

function handleResetPrompt() {
  const config = project.style_config!;
  // 重新从选项合成
  const { stylePrompt, negativePrompt } = compileStylePrompt(config);
  config.compiled_style_prompt = stylePrompt;
  config.compiled_negative_prompt = negativePrompt;
  config.prompt_manually_edited = false;
  wfUpdateProject(project.id, { style_config: config });
}
```

#### 校验规则

| 校验 | 处理 |
|------|------|
| 提示词为空 | 阻止确认，提示「提示词不能为空」 |
| 提示词过长（>2000 字符） | 警告「提示词过长可能影响生成质量」 |
| 重新选择风格/类型时 | 弹窗确认「你已手动编辑过提示词，重新选择将覆盖，是否继续？」 |
```

## 九、版本历史

### 9.1 触发条件

| 操作 | 是否存档 | 快照内容 |
|------|---------|---------|
| 重新选择风格/类型 | ✅ 自动存档 | `style_config` 完整对象 |
| 手动编辑提示词 | ✅ 自动存档 | `style_config` 完整对象 |
| 首次设置参数 | ❌ 不存档 | — |

### 9.2 版本存储

```
storage/projects/proj-xxx/versions/step2/
├── v001_20260328T100000.json    # 首次选吉卜力风格
└── v002_20260328T103000.json    # 改为赛璐璐风格
```

每个版本快照包含完整的 `style_config`，最多保留 **10 个版本**。

### 9.3 回退说明

- 从版本历史恢复 `style_config` 后，项目状态回退到 `configured`
- 如果已完成 Step 3（剧本解析），恢复后需提示用户：「风格已变更，建议重新执行剧本解析」
- 用户可选择保留当前解析结果或重新解析

> 版本历史通用机制详见 `00-工作流总览与集成指南.md` §五。
