## 二、角色三视图生成

### 2.1 生成逻辑

对每个角色，从 `ScriptAnalysis.characters[i].three_view_prompts` 取出三个视图的提示词，调用图像生成 API。

**现有实现**: `workflow-engine.ts` → `generateCharacterViews()` 已存在

需要优化的地方：

```typescript
export async function generateCharacterViews(
  character: ScriptAnalysis["characters"][0],
  projectId: string,
  styleConfig: StyleConfig,          // 🆕 注入风格上下文
  imageModel: string = "nano-banana-2",
): Promise<{ front: string | null; side: string | null; back: string | null }> {

  const views: Record<string, string | null> = { front: null, side: null, back: null };

  for (const view of ["front", "side", "back"] as const) {
    let prompt = character.three_view_prompts[view];
    if (!prompt) continue;

    // 🆕 注入风格关键词（如果 AI 生成的提示词中遗漏）
    if (!prompt.includes(styleConfig.compiled_style_prompt.split(",")[0])) {
      prompt += `, ${extractTopKeywords(styleConfig.compiled_style_prompt, 5)}`;
    }

    try {
      const result = await wfGenerateImage({
        model: imageModel,
        prompt,
        n: 1,                              // 每视图生成 1 张
        size: "1024x1536",                  // 竖版，适合全身
        project_id: projectId,
        asset_filename: `${character.char_id}_${view}.png`,
      });

      views[view] = result.saved_assets?.[0]?.asset_url
        || result.data?.[0]?.url
        || null;
    } catch (err) {
      console.error(`生成 ${character.name} ${view} 视图失败:`, err);
    }
  }

  return views as { front: string | null; side: string | null; back: string | null };
}
```

### 2.2 表情图生成（新增）

除三视图外，为每个主角生成表情图：

```typescript
export async function generateCharacterExpressions(
  character: ScriptAnalysis["characters"][0],
  projectId: string,
  imageModel: string = "nano-banana-2",
): Promise<Record<string, string | null>> {

  const expressions: Record<string, string | null> = {};
  const keys = ["neutral", "happy", "angry", "sad", "surprised", "determined"];

  for (const expr of keys) {
    const prompt = character.expression_prompts?.[expr];
    if (!prompt) continue;

    try {
      const result = await wfGenerateImage({
        model: imageModel,
        prompt,
        n: 1,
        size: "512x512",         // 表情图用方形小尺寸
        project_id: projectId,
        asset_filename: `${character.char_id}_expr_${expr}.png`,
      });

      expressions[expr] = result.saved_assets?.[0]?.asset_url
        || result.data?.[0]?.url
        || null;
    } catch (err) {
      console.error(`生成 ${character.name} ${expr} 表情失败:`, err);
    }
  }

  return expressions;
}
```

## 三、场景六视图生成

### 3.1 生成逻辑

```typescript
export async function generateSceneViews(
  scene: ScriptAnalysis["scenes"][0],
  projectId: string,
  styleConfig: StyleConfig,
  imageModel: string = "nano-banana-2",
): Promise<Record<string, string | null>> {

  const views: Record<string, string | null> = {};
  const viewKeys = ["front", "back", "left", "right", "top", "detail"];

  for (const view of viewKeys) {
    let prompt = scene.six_view_prompts?.[view];
    if (!prompt) continue;

    // 注入风格
    prompt += `, ${extractTopKeywords(styleConfig.compiled_style_prompt, 5)}`;

    try {
      const result = await wfGenerateImage({
        model: imageModel,
        prompt,
        n: 1,
        size: "1536x1024",           // 横版，适合场景全景
        project_id: projectId,
        asset_filename: `${scene.scene_id}_${view}.png`,
      });

      views[view] = result.saved_assets?.[0]?.asset_url
        || result.data?.[0]?.url
        || null;
    } catch (err) {
      console.error(`生成场景 ${scene.name} ${view} 视图失败:`, err);
    }
  }

  return views;
}
```

## 四、并行生成策略

### 4.1 生成队列管理

```typescript
interface AssetGenerationQueue {
  total: number;        // 总任务数
  completed: number;    // 已完成数
  failed: number;       // 失败数
  current: string;      // 当前正在生成的资产名

  // 并发控制
  maxConcurrent: 3;     // 最大并发数
}

async function generateAllAssets(project: WfProject): Promise<void> {
  const tasks: (() => Promise<void>)[] = [];

  // 收集所有角色三视图任务
  for (const char of project.script.analysis!.characters) {
    tasks.push(() => generateCharacterViews(char, project.id, project.style_config!));
    if (char.role === "主角") {
      tasks.push(() => generateCharacterExpressions(char, project.id));
    }
  }

  // 收集所有场景六视图任务
  for (const scene of project.script.analysis!.scenes) {
    tasks.push(() => generateSceneViews(scene, project.id, project.style_config!));
  }

  // 并发执行（最多 3 个并行）
  await parallelLimit(tasks, 3, (completed, total) => {
    // 进度回调 → 更新 UI
  });
}

async function parallelLimit<T>(
  tasks: (() => Promise<T>)[],
  limit: number,
  onProgress?: (completed: number, total: number) => void,
): Promise<T[]> {
  const results: T[] = [];
  let completed = 0;
  let index = 0;

  async function worker(): Promise<void> {
    while (index < tasks.length) {
      const i = index++;
      results[i] = await tasks[i]();
      completed++;
      onProgress?.(completed, tasks.length);
    }
  }

  await Promise.all(Array.from({ length: Math.min(limit, tasks.length) }, worker));
  return results;
}
```

## 五、用户审核

每个角色/场景展示为卡片，卡片内显示所有视图缩略图：

```
┌─ 角色: 小樱 ──────────────────────────────────┐

│  │  正面   │  │  侧面   │  │  背面   │       │

│  └─────────┘  └─────────┘  └─────────┘       │
│                                                │
│  表情: 😐 😊 😡 😢 😲 💪                      │
```

### 5.2 用户操作

| 操作 | 说明 |
|------|------|
| ✅ **通过** | 单张图片标记为通过 |
| 🔄 **重新生成** | 重新调用 API 生成该视图（最多重试 3 次） |
| 📤 **上传替代** | 用户上传自己的图片替代 AI 生成 |
| ✏️ **编辑提示词** | 修改该视图的提示词后重新生成 |
| 🗑️ **删除** | 删除某个角色/场景 |
| 🔒 **锁定** | 全部审核通过后锁定 |

### 5.3 重新生成逻辑

```typescript
async function regenerateView(
  type: "character" | "scene",
  entityId: string,
  viewKey: string,
  customPrompt?: string,    // 用户修改的提示词
  projectId: string,
): Promise<string | null> {
  const prompt = customPrompt || getOriginalPrompt(type, entityId, viewKey);

  const result = await wfGenerateImage({
    model: "nano-banana-2",
    prompt,
    n: 1,
    size: type === "character" ? "1024x1536" : "1536x1024",
    project_id: projectId,
    asset_filename: `${entityId}_${viewKey}_v2.png`,   // 版本后缀
  });

  return result.saved_assets?.[0]?.asset_url || result.data?.[0]?.url || null;
}
```

## 六、资产锁定

### 6.1 锁定条件

所有角色和场景必须满足：
- 至少正面视图已生成且通过审核
- 推荐：三视图/六视图全部通过

### 6.2 锁定后保存

```typescript
async function lockAssets(project: WfProject): Promise<void> {
  // 更新每个角色状态
  for (const char of project.characters) {
    char.status = "done";
  }
  for (const scene of project.scenes) {
    scene.status = "done";
  }

  // 更新项目状态
  await wfUpdateProject(project.id, {
    characters: project.characters,
    scenes: project.scenes,
    status: "assets_locked" as any,
  });
}
```

### 6.3 锁定后不可修改

资产锁定后：
- 角色/场景的视图图片 **不可修改**（除非解锁回退）
- 提示词被固定，作为后续分镜生成的参考
- 开始 Step 5 之前可以「解锁」回到 Step 4 重新编辑

## 九、版本历史

### 9.1 触发条件

| 操作 | 是否存档 | 快照内容 |
|------|---------|---------|
| 单张视图重新生成 | ✅ 自动存档 | 该角色/场景的所有视图 URL |
| 上传替代图 | ✅ 自动存档 | 该角色/场景的所有视图 URL |
| 编辑提示词后重新生成 | ✅ 自动存档 | 该角色/场景的所有视图 URL |
| 「重新生成全部」 | ✅ 自动存档 | 完整 `characters` + `scenes` |
| 首次生成 | ❌ 不存档 | — |

### 9.2 版本存储

```
storage/projects/proj-xxx/versions/step4/
├── v001_20260328T120000.json     # 快照（JSON 引用）
├── v001_assets/                  # 图片副本
│   ├── char_001_front.png
│   ├── char_001_side.png
│   └── scene_001_front.png
├── v002_20260328T123000.json
└── v002_assets/
```

**重要**: Step 4 的版本快照需要**复制图片文件到版本目录**，因为重新生成会覆盖 `assets/` 中的同名文件。这会占用额外磁盘空间，因此版本上限设为 **5 个**。

### 9.3 版本预览信息

```
v002  03-28 12:30  重新生成角色「小樱」侧面视图
  3 角色(9张视图), 4 场景(24张视图)  |  总计 33 张图
```

### 9.4 回退说明

- 恢复版本时，从版本目录复制图片回 `assets/`，覆盖当前文件
- 恢复后资产状态回到「未锁定」，需要重新审核确认
- 如果已完成 Step 5（分镜生成），恢复后提示：「资产已变更，已生成的分镜图/视频可能与新资产不一致」

> 版本历史通用机制详见 `00-工作流总览与集成指南.md` §五。
