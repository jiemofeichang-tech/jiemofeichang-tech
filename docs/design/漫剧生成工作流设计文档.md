## 二、Step 1 — 新建项目

### 2.1 用户输入

| 字段 | 类型 | 说明 |
|------|------|------|
| 项目名称 | 文本 | 可选，不填则自动从剧本摘要生成 |
| 剧本文本 | 长文本 | 支持 200~50000 字，支持粘贴或上传 `.txt` / `.docx` |
| 参考图片 | 图片（可选） | 角色参考图、风格参考图，最多 5 张 |

### 2.2 剧本格式建议

提供模板提示，引导用户按以下结构输入：
```
【故事背景】时代、地点、世界观
【主要角色】角色名 + 外貌 + 性格
【剧情梗概】起承转合
【对白片段】（可选）关键对话
```

系统也应能处理**非结构化**的自由文本输入。

## 三、Step 2 — 风格定调与参数配置

### 3.1 故事短片类型

|------|------|------|-------------------|

### 3.4 风格定调提示词合成逻辑

系统将用户选择 + 内部提示词组合成**总风格提示词**，传递给后续所有生成环节：

```python
def build_style_prompt(user_choices: dict) -> str:
    """
    合成总风格提示词

    输入:
      - story_type: 故事短片类型
      - art_style: 风格大类
      - art_substyle: 风格子类
      - aspect_ratio: 影片比例
      - duration: 时长
      - language: 对白语言
    """

    # 1. 短片类型提示词
    type_prompt = STORY_TYPE_PROMPTS[user_choices["story_type"]]

    # 2. 风格子类提示词（包含最详细的视觉指令）
    style_prompt = STYLE_PROMPTS[user_choices["art_style"]][user_choices["art_substyle"]]

    # 3. 全局质量锚定词（内部固定，用户不可见）
    quality_anchor = (
        "masterpiece quality, professional animation production, "
        "consistent art style throughout all shots, "
        "cinematic composition, professional color grading"
    )

    # 4. 比例相关构图指令
    composition = ASPECT_RATIO_PROMPTS[user_choices["aspect_ratio"]]
    # 例: 9:16 → "vertical composition, portrait framing, mobile-optimized layout"
    # 例: 16:9 → "widescreen cinematic composition, horizontal storytelling"

    # 5. 组合
    return f"""
## 视觉风格总纲
{type_prompt}
{style_prompt}
{quality_anchor}
{composition}

## 项目参数
- 目标时长: {user_choices["duration"]}秒
- 画面比例: {user_choices["aspect_ratio"]}
- 对白语言: {user_choices["language"]}
- 单镜头时长: {user_choices.get("shot_duration", 5)}秒
- 预计分镜数: {user_choices["duration"] // user_choices.get("shot_duration", 5)}
"""
```

## 四、Step 3 — 剧本解析与结构化拆解

### 4.1 AI 剧本解析 Agent

将用户剧本 + 总风格提示词发送给大模型，执行结构化拆解。

#### 4.1.1 系统提示词

```
你是一位专业的动画编剧和分镜导演。你的任务是将用户提供的剧本文本解析为
结构化的动画制作方案。

你必须输出以下内容（JSON 格式）：

1. **剧本摘要** (summary)
   - 一句话概括故事主线（≤50字）
   - 故事基调（喜剧/悲剧/悬疑/热血/温馨/恐怖）
   - 目标受众

2. **角色列表** (characters)
   - 角色名
   - 角色类型（主角/配角/路人）
   - 外貌描述（发型、发色、瞳色、身高体型、年龄段）
   - 服装描述（日常/战斗/礼服等不同状态）
   - 性格标签（3~5个关键词）
   - 角色三视图提示词（英文，用于后续生成）

3. **场景列表** (scenes)
   - 场景名
   - 场景类型（室内/室外/幻想空间）
   - 环境描述（时间、天气、光线、氛围）
   - 场景关键道具
   - 场景六视图提示词（英文，用于后续生成）

4. **分镜脚本** (storyboard)
   - 每个镜头包含：
     - 镜号 (shot_id)
     - 时长 (duration_sec)
     - 景别 (shot_size): 远景/全景/中景/近景/特写
     - 机位角度 (camera_angle): 平视/仰视/俯视/侧视/主观视角
     - 运镜方式 (camera_movement): 固定/推/拉/摇/移/跟/升降/环绕
     - 画面描述 (visual_description): 详细的画面内容（中文）
     - 画面提示词 (visual_prompt): 精确的英文绘画提示词
     - 出场角色 (characters): 角色名列表
     - 所在场景 (scene): 场景名
     - 角色动作 (actions): 角色动态描述
     - 角色表情 (expressions): 情绪状态
     - 对白 (dialogue): 对白文本（如有）
     - 音效 (sfx): 环境音/特效音描述
     - 转场方式 (transition): 切/淡入淡出/划变/闪白

5. **道具列表** (props)（可选）
   - 道具名、外观描述、所属场景

注意事项：
- 分镜拆分标准：场景变化、时间跳转、人物构成变化、情绪转折、
  动作开始/结束、对话主体转换 —— 任一满足即新建镜头
- 一个镜头只讲述一件事，不要在一个镜头中塞入过多信息
- 景别与内容匹配：特写不描述大环境，远景不要求微表情
- 所有英文提示词必须包含当前项目的风格总纲关键词
```

### 4.2 用户审核与编辑

AI 返回结果后，用户可以：

- ✏️ **编辑剧本摘要** — 调整故事基调
- ✏️ **编辑角色** — 修改外貌描述、添加/删除角色
- ✏️ **编辑场景** — 调整场景描述
- ✏️ **编辑分镜** — 调整镜头顺序、修改画面描述、增删镜头

- ✅ **确认** — 锁定当前方案，进入资产生成阶段

### 5.4 资产锁定（Asset Lock）

用户确认所有角色和场景后，系统将资产锁定：

```python
class AssetLibrary:
    """全局资产记忆库 — 确保跨镜头一致性"""

    characters: dict[str, CharacterAsset]  # 角色名 → 三视图 + 特征向量
    scenes: dict[str, SceneAsset]          # 场景名 → 六视图 + 特征向量
    props: dict[str, PropAsset]            # 道具名 → 参考图

    def get_reference_for_shot(self, shot: Shot) -> dict:
        """为单个分镜提取所需的参考资产"""
        return {
            "character_refs": [self.characters[c] for c in shot.characters],
            "scene_ref": self.scenes[shot.scene],
            "prop_refs": [self.props[p] for p in shot.props if p in self.props],
        }
```

## 六、Step 5 — 分镜视频生成与合成

### 6.1 分镜图生成

对每个分镜，系统组合以下元素生成静态分镜图：

```python
def build_shot_prompt(shot: Shot, assets: AssetLibrary, style: str) -> str:
    """
    为单个分镜构建完整的图像生成提示词
    """
    refs = assets.get_reference_for_shot(shot)

    prompt = f"""
    {shot.visual_prompt}

    Shot composition:
    - Shot size: {shot.shot_size}
    - Camera angle: {shot.camera_angle}
    - Camera movement: {shot.camera_movement} (start frame)
    - Lighting: consistent with scene reference

    Character action: {shot.actions}
    Character expression: {shot.expressions}

    {style}
    """

    # 附带参考图（IP-Adapter / ControlNet / Reference-only）
    reference_images = {
        "character_refs": [ref.front_view for ref in refs["character_refs"]],
        "scene_ref": refs["scene_ref"].best_matching_view(shot.camera_angle),
    }

    return prompt, reference_images
```

#### 角色一致性保障

| 技术 | 说明 |
|------|------|
| **IP-Adapter** | 注入角色三视图的特征向量，强制保持人物外观 |
| **ControlNet-Pose** | 通过骨骼姿态控制角色动作 |
| **Reference-Only** | 将场景六视图作为背景参考 |
| **人脸特征锁定** | 提取角色正面图的面部 embedding，每帧注入 |

### 6.2 分镜视频生成

将静态分镜图 + 运镜指令输入视频生成模型：

```python
def generate_shot_video(
    shot: Shot,
    shot_image: Image,       # 分镜起始帧
    end_image: Image | None,  # 分镜结束帧（可选，用于首尾帧模式）
    style_prompt: str,
    duration: int = 5,
) -> Video:
    """
    生成单个分镜的视频片段

    支持的生成模式:
    - text_to_video: 纯文字描述生成
    - image_to_video: 起始帧 → 视频
    - first_last_frame: 首帧 + 尾帧 → 视频（运镜更可控）
    """

    video_prompt = f"""
    {shot.visual_prompt}

    Camera movement: {shot.camera_movement}
    Duration: {duration} seconds
    Character animation: {shot.actions}
    Facial expression: {shot.expressions}

    {style_prompt}
    """

    # 调用视频生成 API（如 Seedance 2.0、Kling、Runway）
    return video_api.generate(
        prompt=video_prompt,
        image=shot_image,
        end_image=end_image,
        duration=duration,
        aspect_ratio=project.aspect_ratio,
    )
```

### 6.3 生成策略

```
分镜 1 ──► 生成起始帧 ──► 图生视频 ──► 5s 片段
分镜 2 ──► 生成起始帧 ──► 图生视频 ──► 5s 片段

```

**并行策略**: 无依赖关系的分镜可并行生成，有前后帧连贯要求的需串行。

### 6.4 用户逐镜审核

每个分镜视频生成后，用户可以：

- ✅ **通过** — 该镜头锁定
- 🔄 **重新生成** — 自动重试（最多 3 次）
- ✏️ **修改提示词** — 调整后重新生成
- 🗑️ **删除** — 移除该镜头

### 6.5 视频合成

所有分镜视频通过后，进入最终合成：

```python
def assemble_final_video(shots: list[ShotVideo], project: Project) -> FinalVideo:
    """
    将所有分镜视频拼接为完整影片
    """

    timeline = Timeline()

    for i, shot_video in enumerate(shots):
        # 1. 添加视频片段
        timeline.add_clip(shot_video.video, start=shot_video.start_time)

        # 2. 添加转场效果
        if i < len(shots) - 1:
            transition_type = shots[i].transition  # 切/淡入淡出/划变/闪白
            timeline.add_transition(transition_type, duration=0.5)

        # 3. 添加对白字幕
        if shot_video.dialogue:
            timeline.add_subtitle(
                text=shot_video.dialogue,
                start=shot_video.start_time,
                duration=shot_video.duration,
                language=project.language,
            )

        # 4. 添加音效
        if shot_video.sfx:
            timeline.add_sfx(shot_video.sfx, start=shot_video.start_time)

    # 5. 添加背景音乐（AI 根据情绪自动配乐）
    bgm = generate_bgm(
        mood=project.summary.tone,
        duration=timeline.total_duration,
        style=project.music_style,
    )
    timeline.add_bgm(bgm, volume=0.3)

    # 6. AI 配音（如果有对白）
    if project.language != "无对白":
        for shot in shots:
            if shot.dialogue:
                voiceover = tts_generate(
                    text=shot.dialogue,
                    character_voice=shot.characters[0].voice_id,
                    language=project.language,
                    emotion=shot.expressions,
                )
                timeline.add_voiceover(voiceover, start=shot.start_time)

    # 7. 渲染输出
    return timeline.render(
        resolution=project.resolution,
        fps=24,
        format="mp4",
        codec="h264",
    )
```

### 6.6 合成后处理

| 处理步骤 | 说明 |
|---------|------|
| 自动调色 | 确保跨镜头色彩一致性 |
| 音频混缩 | 对白 + 音效 + BGM 音量平衡 |
| 字幕烧录 | 可选：硬字幕 / 软字幕(SRT) |
| 片头片尾 | 可选：添加标题卡和演职员表 |

## 七、提示词系统总结

### 7.1 提示词层级架构

```
Layer 0: 全局质量锚定词（固定，用户不可见）
  ↓ 注入到所有生成请求
Layer 1: 短片类型提示词（用户选择触发）
  ↓ 定义叙事节奏和结构
Layer 2: 风格大类 + 子类提示词（用户选择触发）
  ↓ 定义视觉风格
Layer 3: 角色/场景专属提示词（AI 从剧本生成）
  ↓ 定义具体对象外观
Layer 4: 分镜提示词（AI 从剧本生成）
  ↓ 定义每个镜头的画面
Layer 5: 反面提示词（内部匹配，用户不可见）
  ↓ 排除不良特征
```

### 7.2 提示词合成公式

```
最终分镜提示词 =
  分镜画面描述 (Layer 4)
  + 角色外观关键词 (Layer 3)
  + 场景环境关键词 (Layer 3)
  + 风格子类提示词 (Layer 2)
  + 短片类型提示词 (Layer 1)
  + 质量锚定词 (Layer 0)

最终反面提示词 =
  风格子类反面词 (Layer 5)
  + 通用质量反面词 ("low quality, blurry, deformed, extra limbs, bad anatomy")
```

## 八、七大 AI Agent 分工

参考 OiiOii 的多 Agent 协作架构，定义以下 Agent：

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **编剧 Agent** | 剧本解析、结构化拆解 | 用户剧本 + 风格定调 | 剧本JSON（摘要/角色/场景/分镜） |
| **艺术总监 Agent** | 风格把控、提示词优化 | 风格选择 + 剧本 | 优化后的全局风格提示词 |
| **角色设计师 Agent** | 角色三视图生成 | 角色描述 + 风格 | 角色参考图 + 特征向量 |
| **场景设计师 Agent** | 场景六视图生成 | 场景描述 + 风格 | 场景参考图 + 空间结构 |
| **分镜师 Agent** | 分镜图 + 分镜视频生成 | 分镜脚本 + 资产库 | 分镜视频片段 |
| **音效总监 Agent** | 配乐 + 配音 + 音效 | 情绪基调 + 对白 | 音频轨道 |
| **合成导演 Agent** | 最终视频合成 + 质量检查 | 所有分镜 + 音频 | 完整 MP4 |

## 十、提示词优化建议

### 10.1 当前设计中需注意的问题

| 问题 | 风险 | 建议 |
|------|------|------|
| 提示词过长 | token浪费、模型注意力分散 | 分层注入，每层有 token 预算上限 |
| 中英文混用 | 生图模型对英文理解更好 | 面向模型的提示词统一用英文 |
| 风格冲突 | 用户选"水墨风"但剧本是赛博朋克 | AI 检测冲突并提示用户 |
| 角色描述模糊 | 三视图不一致 | 强制要求量化描述（发色、瞳色、身高等具体值） |
| 分镜过密 | 5秒内塞太多动作 | AI 自动检查并拆分过密镜头 |

### 10.2 提示词质量保障机制

```python
class PromptValidator:
    """提示词质量检查器"""

    def check_style_conflict(self, style: str, script: str) -> list[Warning]:
        """检测风格与剧本内容的冲突"""

    def check_shot_density(self, shot: Shot) -> list[Warning]:
        """检测单镜头信息密度是否过高"""

    def check_character_consistency(self, char_prompt: str, style_prompt: str) -> list[Warning]:
        """检测角色提示词是否与风格兼容"""

    def check_negative_prompt_conflict(self, positive: str, negative: str) -> list[Warning]:
        """检测正面和反面提示词是否矛盾"""
```
